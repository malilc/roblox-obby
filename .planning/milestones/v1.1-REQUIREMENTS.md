# Requirements: Mobile HUD Fix

**Defined:** 2026-03-21
**Core Value:** Every UI element must be fully visible, usable, and non-overlapping on mobile

## v1.1 Requirements

### Menu Grid

- [x] **MENU-01**: Menu grid (left side) scales down on mobile to not dominate the screen
- [x] **MENU-02**: Menu grid buttons remain tappable at reduced size (44px+ hit area)

### Item Slots

- [ ] **ITEM-01**: Item slots render only once on mobile (fix duplication bug -- desktop is fine)
- [x] **ITEM-02**: Item bar positioned at bottom edge of screen (not floating)

### HUD Overlap

- [ ] **HUD-01**: Rankings panel does not overlap item slots or jump button on mobile
- [ ] **HUD-02**: Coins/Gems bar does not overlap walk/movement joystick on mobile
- [ ] **HUD-03**: All HUD elements remain non-overlapping on desktop (no regression)

## v2 Requirements

### Advanced HUD

- **HUD-04**: HUD elements auto-reposition based on viewport size (responsive layout)
- **HUD-05**: HUD transparency/visibility toggles for mobile gameplay

## Out of Scope

| Feature | Reason |
|---------|--------|
| Modal scaling | Already complete in v1.0 |
| Leaderboard billboard sizing | 3D world objects, not HUD |
| New HUD features or redesign | Purely fixing overlaps and sizing |
| Touch control customization | Roblox default controls, not our code |
