# Phase 1 — UI Design Contract

> Visual and interaction contract for the responsive scaling infrastructure and 4 small modal integrations. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Roblox Luau — no web component library) |
| Preset | not applicable |
| Component library | UIFactory.luau (project-internal) + ThemeConfig.luau (design tokens) |
| Icon library | Unicode emoji (inline in TextLabel/TextButton `.Text`) |
| Font | Gotham family (Roblox built-in: Gotham, GothamMedium, GothamBold, GothamBlack) |

**Note:** This is a Roblox game UI project. All UI is created programmatically in Luau via Instance creation. There is no HTML/CSS, no npm, no shadcn. The design system is ThemeConfig.luau (color/structure tokens) + UIFactory.luau (component builders).

## Spacing Scale

Declared values from existing ThemeConfig.luau (source of truth):

| Token | Value | Usage |
|-------|-------|-------|
| PAD_SM | 6px | Compact element spacing, inset padding |
| PAD_MD | 10px | Default element spacing |
| PAD_LG | 16px | Section padding, modal internal padding |
| Modal padding (D-04) | 40px per side (80px total) | Viewport margin around scaled modals on mobile |

Exceptions:
- The 40px modal viewport padding (D-04) is a scaling-specific constant living in ResponsiveScale.luau, NOT in ThemeConfig. It is not a general spacing token — it is the gap between screen edge and modal frame on mobile viewports.
- The 8px gap between toggle rows in SettingsUI is inline (ROW_H + 8). Not tokenized, acceptable for Phase 1.

**Scaling contract:** UIScale does NOT change any spacing values. All existing PAD_SM/PAD_MD/PAD_LG values remain as-is in code. UIScale uniformly shrinks the rendered output including all padding.

## Typography

Existing typography from the 4 in-scope modals (SettingsUI, FeedbackUI, SpectatorUI, LeaderboardUI):

| Role | Size | Weight (Enum.Font) | Line Height |
|------|------|---------------------|-------------|
| Modal title | 22px | GothamBlack (900) | Engine default (1.0) |
| Body / input text | 16px | Gotham (400) / GothamBold (700) | Engine default (1.0) |
| Label / secondary | 13px | GothamBold (700) / GothamMedium (500) | Engine default (1.0) |
| Caption / meta | 11-12px | Gotham (400) / GothamMedium (500) | Engine default (1.0) |

**Scaling contract:**
- At scale 1.0 (desktop): All text sizes render at their declared pixel values. No change from current behavior.
- At scale 0.835 (414px phone, 400px-wide control bar): 22px title becomes ~18px effective, 16px body becomes ~13px effective, 13px label becomes ~11px effective, 11px caption becomes ~9px effective.
- At scale 0.5 (floor): 22px title becomes 11px effective, 16px body becomes 8px effective. This is the absolute minimum — below this, text becomes unreadable, which is why D-05 sets 0.5 as the floor.
- Roblox TextLabel does not expose a line-height property. Line height is engine-controlled at ~1.0x font size.

**Minimum readable sizes after scaling (validation targets):**
- Modal titles must remain above 11px effective (readable at arm's length on mobile)
- Body text must remain above 8px effective (absolute minimum for Roblox text rendering)
- The 0.5 floor guarantees these minimums for the small modals (360px design width)

## Color

All colors are defined in ThemeConfig.luau. Phase 1 does NOT introduce any new colors. The scaling infrastructure (UIScale) is a render-level transform that does not affect color values.

| Role | Value (RGB) | ThemeConfig Token | Usage in Phase 1 modals |
|------|-------------|-------------------|------------------------|
| Dominant (60%) | rgb(195, 225, 245) | BG_BASE | Modal container backgrounds (SettingsUI, FeedbackUI, SpectatorUI panels) |
| Secondary (30%) | rgb(255, 255, 255) | BG_SURFACE | Toggle backgrounds, input fields, button surfaces, card backgrounds |
| Accent — primary CTA | rgb(75, 200, 140) | ACCENT_CYAN | SpectatorUI "SPECTATE" button, glow strokes on spectator panels |
| Accent — success | rgb(80, 230, 120) | SUCCESS | Toggle "on" state (SettingsUI), Submit button (FeedbackUI) |
| Destructive | rgb(255, 70, 70) | DANGER | "LEAVE" button text (SpectatorUI), category button (FeedbackUI bug) |
| Text primary | rgb(55, 60, 72) | TEXT_PRIMARY | All primary text on light backgrounds |
| Text secondary | rgb(70, 78, 95) | TEXT_MUTED | Close buttons, secondary labels, ranking entries |
| Elevated surface | rgb(235, 240, 248) | BG_ELEVATED | Separators, control bar buttons, hover states |

Accent reserved for: SpectatorUI "SPECTATE" button + panel glow strokes (ACCENT_CYAN), toggle "on" state + submit button (SUCCESS). Never applied to backgrounds or passive elements.

**Scaling contract:** UIScale does not alter BackgroundColor3, TextColor3, UIStroke.Color, or UIGradient.Color values. All colors render identically at any scale factor.

## Interaction Contract

Phase 1 adds one new interaction behavior (viewport-responsive scaling) while preserving all existing interactions.

### Scale Behavior (NEW)

| Property | Value | Source |
|----------|-------|--------|
| Scale formula | `math.min(1.0, (viewportW - 80) / designW, (viewportH - guiInset - 80) / designH)` | D-06, RESEARCH.md |
| Minimum scale floor | 0.5 | D-05 |
| Maximum scale cap | 1.0 (never upscale) | SCALE-05 |
| Viewport padding | 40px per side (80px total) | D-04 |
| Resize response | Recalculate on Camera.ViewportSize change, after 1-frame yield | SCALE-03, RESEARCH.md |
| Animation on resize | None — instant scale update (no tween) | Phase 1 default; animation is Phase 3 POLISH-01 |

### Expected Scale Values by Viewport

| Viewport | Modal | Scale Factor | Effective Modal Size |
|----------|-------|-------------|---------------------|
| 1920x1080 (desktop) | SettingsUI 360x380 | 1.0 | 360x380 (unchanged) |
| 1920x1080 (desktop) | SpectatorUI controlBar 400x44 | 1.0 | 400x44 (unchanged) |
| 414x736 (iPhone SE landscape-ish) | SettingsUI 360x380 | 0.928 (min of (334/360, 656/380)) | 334x353 |
| 414x736 (iPhone SE landscape) | SpectatorUI controlBar 400x44 | 0.835 (334/400) | 334x37 |
| 375x667 (iPhone 8 landscape) | SettingsUI 360x380 | 0.819 (min of (295/360, 587/380)) | 295x311 |

**Note:** Actual viewportH depends on GuiInset (~36-58px), which reduces available height. Values above assume ~80px GuiInset.

### Existing Interactions (PRESERVED — no changes)

| Interaction | Modal | Behavior |
|------------|-------|----------|
| Toggle tap | SettingsUI | Knob tweens 0.15s, background color tweens 0.15s |
| Close button tap | SettingsUI, FeedbackUI | Hides modal immediately (Visible = false) |
| Category select | FeedbackUI | Button color tweens 0.12s to selected/deselected state |
| Text input | FeedbackUI | Character count updates live, capped at 500 chars |
| Submit | FeedbackUI | Button grays out + text changes to "Sending...", re-enables on response |
| Spectate prompt | SpectatorUI | Slides up with Back easing over 0.3s |
| Player cycling | SpectatorUI | Prev/Next buttons cycle through active player list |
| Mode toggle | SpectatorUI | Swaps between Follow and FreeCam, updates button text |

**Scaling does NOT affect interaction behavior.** UIScale is a visual transform only — all click/touch hit areas scale proportionally with the visual elements. A 32x32 close button at 0.835x scale becomes a 27x27 effective hit area (above the 24px minimum for Roblox, acceptable for Phase 1; 44px touch target enforcement is Phase 3 TOUCH-01/02).
