    Create `src/shared/ResponsiveScale.luau` with this exact structure (per D-01, D-08, RESEARCH.md Pattern 1):

    ```luau
    local ResponsiveScale = {}

    local MIN_SCALE = 0.5       -- per D-05
    local PADDING = 80          -- per D-04, D-06: 40px each side

    local trackedScales: {{ uiScale: UIScale, designWidth: number, designHeight: number }} = {}

    -- Pure function: testable without Roblox runtime services
    -- (viewportW, viewportH, guiInsetY, designW, designH) -> number
    function ResponsiveScale.calculateScale(
        viewportW: number,
        viewportH: number,
        guiInsetY: number,
        designW: number,
        designH: number
    ): number
        local availableW = viewportW - PADDING
        local availableH = viewportH - guiInsetY - PADDING
        local scaleX = availableW / designW
        local scaleY = availableH / designH
        local scale = math.min(scaleX, scaleY, 1.0)
        return math.max(scale, MIN_SCALE)
    end

    -- Runtime wrapper: reads Camera.ViewportSize and GuiInset, then calls calculateScale
    function ResponsiveScale._getViewportScale(designW: number, designH: number): number
        local Camera = workspace.CurrentCamera
        local GuiService = game:GetService("GuiService")
        local viewport = Camera.ViewportSize
        local guiInset = GuiService:GetGuiInset()
        return ResponsiveScale.calculateScale(viewport.X, viewport.Y, guiInset.Y, designW, designH)
    end

    -- Apply UIScale to a frame and register for viewport updates (per D-08)
    function ResponsiveScale.applyToFrame(
        frame: GuiObject,
        designWidth: number,
        designHeight: number
    ): UIScale
        local uiScale = Instance.new("UIScale")
        uiScale.Scale = ResponsiveScale._getViewportScale(designWidth, designHeight)
        uiScale.Parent = frame
        table.insert(trackedScales, {
            uiScale = uiScale,
            designWidth = designWidth,
            designHeight = designHeight,
        })
        return uiScale
    end

    -- Recalculate all tracked UIScale instances (per SCALE-03)
    function ResponsiveScale._updateAll()
        for i = #trackedScales, 1, -1 do
            local entry = trackedScales[i]
            if entry.uiScale.Parent then
                entry.uiScale.Scale = ResponsiveScale._getViewportScale(
                    entry.designWidth,
                    entry.designHeight
                )
            else
                table.remove(trackedScales, i)
            end
        end
    end

    -- Initialize viewport change listener (per SCALE-03, Recommendation 2)
    function ResponsiveScale.init()
        local Camera = workspace.CurrentCamera
        Camera:GetPropertyChangedSignal("ViewportSize"):Connect(function()
            task.wait() -- one frame yield for layout to settle (Recommendation 2)
            ResponsiveScale._updateAll()
        end)
    end

    return ResponsiveScale
    ```

    Key design decisions in this implementation:
    - `calculateScale` is a PURE function with all inputs as parameters -- this is the testable core (per Claude's Discretion: internal data structure + GuiInset handling)
    - `_getViewportScale` is the runtime wrapper that reads Camera/GuiService
    - `applyToFrame` creates UIScale child and registers for updates (per D-08)
    - `_updateAll` uses reverse iteration for safe removal of orphaned entries (per Recommendation 3)
    - `init` sets up the viewport change listener with `task.wait()` (per Recommendation 2)
    - GuiInset subtracted from viewport height (per Recommendation 1, since IgnoreGuiInset=false in MainUI.luau line 60)

    Run tests -- they MUST pass.
