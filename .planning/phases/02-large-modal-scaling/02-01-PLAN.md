  <action>
    Per D-01, D-02, D-03: Extend ResponsiveScale.luau with stroke compensation.

    1. Add a module-level storage table for tracked strokes:
    ```luau
    local trackedStrokes: {{ frame: GuiObject, strokes: {{ stroke: UIStroke, originalThickness: number }} }} = {}
    ```

    2. Add `ResponsiveScale.compensateStrokes(frame: GuiObject, scale: number)`:
       - Use `frame:GetDescendants()` for recursive UIStroke discovery (per Claude's Discretion — flat array, performant for 50-200 element UI hierarchies)
       - For each UIStroke descendant, store `{ stroke = desc, originalThickness = desc.Thickness }` and set `desc.Thickness = desc.Thickness * scale`
       - Check if this frame already has a tracked entry (by reference equality). If so, update strokes list without creating duplicates. If not, insert new entry.
       - Only insert into trackedStrokes if at least 1 UIStroke was found

    3. Update `ResponsiveScale._updateAll()` to recalculate strokes after the existing UIScale loop:
       - Reverse iterate `trackedStrokes`
       - If `entry.frame.Parent` exists, find the UIScale child (`entry.frame:FindFirstChildOfClass("UIScale")`), then for each strokeEntry where `strokeEntry.stroke.Parent` exists, set `strokeEntry.stroke.Thickness = strokeEntry.originalThickness * uiScale.Scale`
       - If `entry.frame.Parent` is nil (orphaned), `table.remove(trackedStrokes, i)` (same cleanup pattern as trackedScales)

    4. Add unit tests to ResponsiveScale.spec.luau in a new `describe("compensateStrokes", ...)` block. Since compensateStrokes operates on Instance objects that require Roblox runtime, write the tests but note they require run-in-roblox execution (same as existing tests that use `game:GetService`).

    Use tab indentation to match existing codebase convention.
  </action>
