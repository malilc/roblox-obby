1. **Add mobile detection (same proven pattern from Phase 4 per D-05):**
   At the end of createUI(), after the existing ResponsiveScale.applyToFrame calls, add a viewport-based mobile layout updater:
   - Get `local camera = workspace.CurrentCamera`
   - Create a separate UIScale instance for the rankings panel mobile scaling (name it `rankMobileScale`). Set initial Scale=1. Parent to rankingsPanel. NOTE: This is IN ADDITION to the existing ResponsiveScale UIScale -- the ResponsiveScale one handles modal-style viewport scaling, this new one handles the extra mobile shrink factor.
   - Actually, SIMPLER approach: Remove the existing `ResponsiveScale.applyToFrame(self.rankingsPanel, RANKINGS_W, RANKINGS_H)` call (line 186) and replace with a manual UIScale that serves BOTH purposes. On mobile set Scale=0.55 (similar to menu grid's 0.55 from Phase 4). On desktop set Scale=1.0 (original size, no modal scaling needed for a HUD element).
   - Create `local function updateRankingsLayout()` that checks `camera.ViewportSize.Y > 1 and camera.ViewportSize.Y < 500` for mobile.
   - On mobile: set rankUIScale.Scale = 0.55, reposition rankings panel to top-left: Position = UDim2.new(0, 6, 0, 50). This keeps it in the top-left corner away from center-bottom item buttons and right-bottom jump button.
   - On desktop: set rankUIScale.Scale = 1.0, restore Position = UDim2.new(0, 10, 0, 60) (original position).
   - Call updateRankingsLayout() immediately, and connect to `camera:GetPropertyChangedSignal("ViewportSize")`.
   - Store `self.isMobile` flag so updateRankings can read it.

2. **Limit rankings to top 3 on mobile (per D-01):**
   In `updateRankings()` method (around line 494), where rankings entries are created in the for loop:
   - Before the loop, add: `local maxEntries = if self.isMobile then 3 else #self.latestRankings`
   - Change the loop from `for i, ranking in ipairs(self.latestRankings) do` to `for i = 1, math.min(#self.latestRankings, maxEntries) do` and get `local ranking = self.latestRankings[i]` inside the loop.
   - This limits visual display to top 3 on mobile while keeping the full data for player tracking.

3. **Also remove the ResponsiveScale.applyToFrame for promptContainer and controlBar** since those are HUD elements, not modals. Replace with the same viewport-based UIScale pattern:
   - For promptContainer: on mobile Scale=0.7, on desktop Scale=1.0 (keeps prompt usable on mobile).
   - For controlBar: on mobile Scale=0.7, on desktop Scale=1.0.
   - Add these to the same updateRankingsLayout() function (rename it to updateMobileLayout() for clarity).
   - Remove lines: `ResponsiveScale.applyToFrame(self.promptContainer, PROMPT_W, PROMPT_H)` and `ResponsiveScale.applyToFrame(controlBar, CONTROL_W, CONTROL_H)`.

4. **Do NOT modify any other file.** Desktop behavior must remain identical.
