<objective>
Fix item slot duplication on mobile so only one set of item controls is visible.

Purpose: On mobile, players see two sets of item slots -- ItemUI creates centered item slots with keybind labels [1][2][3], and MobileInputUI creates separate touch buttons in the bottom-right. This duplication is confusing. The fix hides ItemUI's keybind hint labels on mobile (the [1][2][3] text below each slot) so the visual clutter is reduced, while keeping ItemUI's slots visible for tooltip access and MobileInputUI's buttons as the primary touch interaction.

Output: ItemUI with hidden keybind labels on mobile, MobileInputUI unchanged, visual verification checkpoint.
</objective>

**Per D-06 and D-07: Fix the visual duplication of item controls on mobile.**

The root cause (D-06): ItemUI creates item slot frames with keybind labels ([1], [2], [3]) and MobileInputUI creates separate touch buttons -- both are visible on mobile.

The cleanest approach (Claude's discretion from D-07): Hide ItemUI's keybind hint labels on mobile. The keybind hints reference keyboard keys (1/2/3) which are irrelevant on touch devices. The ItemUI slot frames themselves should remain visible for tooltip access (tapping a slot shows item description). MobileInputUI touch buttons in bottom-right remain as the primary item activation controls.

1. Add `local UserInputService = game:GetService("UserInputService")` at top. Note: UserInputService is already required for setupInput(), so this service is already available. Just add a module-level constant:
```luau
local IS_MOBILE = UserInputService.TouchEnabled
```
Add this after the `local ItemUI = {}` line.

2. In the `createSlot()` method, find the keybind hint creation block (around line 282-293). After the line `keybindHint.Parent = slotFrame`, add:
```luau
	-- Hide keybind hints on mobile (touch devices use MobileInputUI buttons instead)
	if IS_MOBILE then
		keybindHint.Visible = false
	end
```

This is the minimal change needed. The ItemUI slots remain visible and tappable for tooltip display. The MobileInputUI touch buttons remain unchanged as the primary item-use control on mobile. On desktop, keybind hints [1][2][3] remain visible as before.

**Do NOT change:**
- ItemUI slot frame visibility (slots must stay visible for tooltips)
- ItemUI setupInput() keyboard handling (harmless on mobile, keys simply don't exist)
- MobileInputUI in any way (it already works correctly)
- Any item slot sizes, positions, or colors
